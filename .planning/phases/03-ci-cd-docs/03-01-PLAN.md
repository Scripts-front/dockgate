<!-- API contracts extracted from src/routes/apps.ts — executor must use these exactly -->

POST /apps/:name/upload?version=X
  - Header: Authorization: Bearer <UPLOAD_TOKEN>
  - version is a query param (not body)
  - Response: { "url": string }  — presigned PUT URL, no other fields
  - URL expires in 900 seconds

PUT /apps/:name/latest
  - Header: Authorization: Bearer <UPLOAD_TOKEN>
  - Header: Content-Type: application/json
  - Body: { "version": string, "sha256": string, "size": number }
    - size MUST be a JSON integer (not string)
    - sha256 MUST be exactly 64 lowercase hex characters
    - publishedAt is NOT sent — API sets it via new Date().toISOString()
  - Response: { "ok": true }
  - Returns 422 if .tar file not found in MinIO (anti-phantom check)
  - Returns 400 if sha256 or size are invalid

GET /apps/:name/latest
  - No auth required
  - Response: { schema: 1, version, sha256, size, publishedAt }

MinIO object keys:
  - Tar file:      {appName}/{version}.tar
  - Latest index:  {appName}/latest.json
</interfaces>

The YAML must implement ALL of the following (per D-05):

**Trigger (D-03):**

**Top-level env block (Claude's Discretion — use configurable vars):**
```yaml
env:
  APP_NAME: my-app          # Change to your app name registered in DockGate
  IMAGE_NAME: my-org/my-app # Docker image to build (org/name format)
```

**Job: `publish` on `ubuntu-latest`**

**Step 1 — Checkout:**
```yaml
- name: Checkout
  uses: actions/checkout@v4
```

**Step 2 — Extract version (D-04):**
```yaml
- name: Extract version from tag
  id: version
  run: echo "VERSION=${GITHUB_REF#refs/tags/v}" >> $GITHUB_OUTPUT
```
Do NOT use the deprecated `echo "::set-output name=VERSION::$VALUE"` syntax.

**Step 3 — Docker build:**
```yaml
- name: Build Docker image
  run: docker build -t $IMAGE_NAME:${{ steps.version.outputs.VERSION }} .
```

**Step 4 — Docker save (not export):**
```yaml
- name: Export image to tar
  run: docker save $IMAGE_NAME:${{ steps.version.outputs.VERSION }} -o image.tar
```

**Step 5 — Calculate sha256 and size (D-06):**
```yaml
- name: Calculate sha256 and size
  id: integrity
  run: |
    echo "SHA256=$(sha256sum image.tar | awk '{ print $1 }')" >> $GITHUB_OUTPUT
    echo "SIZE=$(stat -c%s image.tar)" >> $GITHUB_OUTPUT
```

**Step 6 — Request upload URL (POST /upload):**
```yaml
- name: Request upload URL from DockGate
  id: upload_url
  env:
    DOCKGATE_URL: ${{ secrets.DOCKGATE_URL }}
    DOCKGATE_UPLOAD_TOKEN: ${{ secrets.DOCKGATE_UPLOAD_TOKEN }}
  run: |
    RESPONSE=$(curl -sf -X POST \
      -H "Authorization: Bearer $DOCKGATE_UPLOAD_TOKEN" \
      "$DOCKGATE_URL/apps/$APP_NAME/upload?version=${{ steps.version.outputs.VERSION }}")
    echo "URL=$(echo $RESPONSE | jq -r .url)" >> $GITHUB_OUTPUT
```

**Step 7 — PUT .tar directly to MinIO (no Authorization header — presigned URL carries credentials):**
```yaml
- name: Upload image to MinIO
  run: |
    curl -sf -X PUT \
      -H "Content-Type: application/octet-stream" \
      --upload-file image.tar \
      "${{ steps.upload_url.outputs.URL }}"
```

**Step 8 — Publish latest via PUT /latest:**
```yaml
- name: Publish latest version via DockGate
  env:
    DOCKGATE_URL: ${{ secrets.DOCKGATE_URL }}
    DOCKGATE_UPLOAD_TOKEN: ${{ secrets.DOCKGATE_UPLOAD_TOKEN }}
  run: |
    curl -sf -X PUT \
      -H "Authorization: Bearer $DOCKGATE_UPLOAD_TOKEN" \
      -H "Content-Type: application/json" \
      -d "{\"version\":\"${{ steps.version.outputs.VERSION }}\",\"sha256\":\"${{ steps.integrity.outputs.SHA256 }}\",\"size\":${{ steps.integrity.outputs.SIZE }}}" \
      "$DOCKGATE_URL/apps/$APP_NAME/latest"
```

CRITICAL: `size` in the JSON body must NOT be quoted (`${{ steps.integrity.outputs.SIZE }}` with no surrounding `"`). The API validates `typeof size === 'number'` and rejects strings with 400.

Include inline YAML comments on each step explaining what it does and why (Claude's Discretion — comments make the file self-documenting for developers unfamiliar with DockGate).

The workflow name (top-level `name:` field) should be: `Build and Publish Docker Image via DockGate`

**4. Pipeline Steps**
Numbered walk-through (1-7) explaining each step. For each step: what it does (1 sentence) + why it matters for DockGate (1-2 sentences for DockGate-specific steps). Steps:
1. Checkout — standard, minimal explanation
2. Extract version from tag — explain: DockGate uses the version string as the MinIO object key (`{appName}/{version}.tar`), so the version must be consistent across the workflow
3. Build Docker image — standard, minimal explanation
4. Export image to tar — explain: must use `docker save` (not `docker export`); `docker save` preserves all layers and metadata; the output is the file uploaded to MinIO
5. Calculate sha256 and size — explain: DockGate's `PUT /latest` requires sha256 (64 lowercase hex chars) and size (bytes, integer) for integrity verification; these are calculated before any API call
6. Request upload URL — explain: `POST /apps/:name/upload?version=X` returns a presigned PUT URL valid for **900 seconds**; the upload (step 7) must complete within this window; the URL carries MinIO credentials — no Authorization header needed for the upload itself
7. Upload image to MinIO — explain: PUT goes directly to MinIO using the presigned URL; DockGate never sees the file bytes (no proxy); `Content-Type: application/octet-stream` is required
8. Publish latest version — explain: `PUT /apps/:name/latest` with `{ version, sha256, size }` atomically updates the latest pointer; the API verifies the `.tar` exists in MinIO before writing (anti-phantom protection); `size` must be a JSON integer, not a string

**5. Complete Workflow**
Include the content of `docs/examples/github-actions.yml` in a fenced code block (`\`\`\`yaml`), OR reference the file with: "See [`docs/examples/github-actions.yml`](examples/github-actions.yml) for the complete workflow."

Use the reference approach (link to file) — avoids duplicating content.

**6. Troubleshooting**
Include these failure modes (derived from 03-RESEARCH.md common pitfalls):

- **403 on MinIO upload (step 7):** Presigned URL expired. The upload URL is valid for 900 seconds. If your build takes more than 15 minutes, the PUT will be rejected. Solution: ensure steps 5→7 are in a single job and run immediately after the build.
- **400 Invalid size:** The `size` field in the PUT /latest body is a string instead of integer. Check that `SIZE` from `stat -c%s` is interpolated without surrounding quotes in the JSON body.
- **400 Invalid sha256:** sha256 format is wrong. `sha256sum` on Linux outputs lowercase hex — this should work as-is. Verify the `awk '{ print $1 }'` extracts only the hash (first field), not the filename.
- **422 Tar file not found:** The PUT /latest was called but the `.tar` upload to MinIO (step 7) did not complete successfully. Check step 7 curl exit code.
- **Double slash in URL:** `DOCKGATE_URL` has a trailing slash. Remove it — the workflow appends `/apps/...` directly.

Write in plain markdown with ATX headers (`##`). Use code blocks for all shell commands and API payloads. Keep a developer tone — skip boilerplate warnings, be direct.

```markdown
# DockGate

Private Docker image distribution API for authorized clients. Stores Docker images exported as `.tar` files in MinIO and serves presigned download URLs — no Docker Registry, no registry login, no complexity.

## What it does

- CI/CD pipelines push new Docker image versions via the DockGate API
- Authorized clients receive a presigned MinIO URL and download the `.tar` directly
- Supports multiple apps, semantic versioning, and integrity verification (sha256 + size)

## Endpoints

| Method | Path | Auth | Description |
|--------|------|------|-------------|
| GET | `/health` | None | Health check (API + MinIO connectivity) |
| GET | `/apps/:name/latest` | None | Latest version metadata |
| GET | `/apps/:name/download?version=X` | DOWNLOAD_TOKEN | Presigned download URL |
| POST | `/apps/:name/upload?version=X` | UPLOAD_TOKEN | Presigned upload URL for CI/CD |
| PUT | `/apps/:name/latest` | UPLOAD_TOKEN | Publish new latest version |

## Configuration

Required environment variables:

| Variable | Description |
|----------|-------------|
| `DOWNLOAD_TOKEN` | Bearer token for client download requests |
| `UPLOAD_TOKEN` | Bearer token for CI/CD upload requests |
| `MINIO_ENDPOINT` | MinIO hostname (internal, no protocol, e.g., `minio`) |
| `MINIO_PUBLIC_ENDPOINT` | MinIO base URL for presigned URLs (externally resolvable, e.g., `http://192.168.1.10:9000`) |
| `MINIO_BUCKET` | MinIO bucket name |
| `MINIO_ACCESS_KEY` | MinIO access key |
| `MINIO_SECRET_KEY` | MinIO secret key |
| `PORT` | (optional) HTTP port, defaults to `3000` |

## CI/CD Integration

See [docs/ci-cd.md](docs/ci-cd.md) for the complete GitHub Actions integration guide with a ready-to-copy workflow YAML.
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Developer → GitHub Actions workflow | Workflow YAML runs in GitHub-controlled runners; secrets injected via GitHub Secrets mechanism |
| GitHub runner → DockGate API | POST /upload and PUT /latest calls carry UPLOAD_TOKEN in Authorization header |
| GitHub runner → MinIO presigned URL | PUT uses URL that already contains MinIO credentials; no additional auth header |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Information Disclosure | YAML `run:` blocks | mitigate | Secrets are passed via `env:` block scoped to the step (not top-level env), preventing exposure in logs. The YAML template must use `${{ secrets.X }}` pattern inside `env:` blocks, never inline in `run:` shell commands via string interpolation. |
| T-03-02 | Tampering | Presigned URL interception | accept | Presigned URL is single-use and short-lived (900s). Man-in-the-middle on HTTPS is out of scope for this threat model. The YAML uses the URL directly; no logging of the URL in the workflow. |
| T-03-03 | Repudiation | No audit of what was published | accept | Out of scope for v1 (SEC-V2-02 covers audit logging). `PUT /latest` is atomic — only one version is "latest" at any time. |
| T-03-04 | Information Disclosure | `DOCKGATE_UPLOAD_TOKEN` in shell env | mitigate | Token is accessed via `$DOCKGATE_UPLOAD_TOKEN` inside scoped `env:` blocks on individual steps. GitHub Actions masks secrets in logs automatically. The YAML template follows this pattern — no token in `run:` string literals. |
| T-03-05 | Elevation of Privilege | Malicious YAML copied by developer | accept | The YAML is a documentation artifact, not executed in this repo. Developers who copy it are responsible for reviewing it. The YAML makes no requests except to the developer's own DockGate instance. |
| T-03-06 | Denial of Service | Presigned URL reuse or replay | accept | Presigned URL expires in 900s. MinIO enforces expiry server-side. No additional mitigation needed in the documentation layer. |
</threat_model>
