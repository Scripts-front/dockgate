**src/routes/health.ts** (INFRA-01):

The health check calls `bucketExists()` on every request to get a live MinIO status reading. Wraps in try/catch because `bucketExists()` throws on network failure — both `false` return and thrown exception must result in 503.

```typescript
// src/routes/health.ts
import { Router } from 'express'
import { minioClient } from '../lib/minio.ts'
import { config } from '../config.ts'

export const healthRouter = Router()

healthRouter.get('/health', async (_req, res) => {
  try {
    const alive = await minioClient.bucketExists(config.minioBucket)
    if (alive) {
      res.json({ ok: true, minio: 'connected' })
    } else {
      res.status(503).json({ ok: false, minio: 'unreachable' })
    }
  } catch {
    res.status(503).json({ ok: false, minio: 'unreachable' })
  }
})
```

**src/index.ts** (INFRA-01, INFRA-03):

Bootstrap sequence — order is critical:
1. Import `'./config.ts'` first (triggers fail-fast validation at import time)
2. Import all modules
3. Create express app and add middleware
4. Add routes
5. Add global error handler (4-parameter Express 5 signature required)
6. `await verifyMinioConnection()` — process.exit(1) if MinIO bucket missing
7. `app.listen(config.port, ...)` — ONLY after step 6 succeeds

**The server must never accept requests before MinIO is verified** (Pitfall 5).

```typescript
// src/index.ts
import './config.ts'           // Step 1: fail-fast env validation runs at import time
import express from 'express'
import { verifyMinioConnection } from './lib/minio.ts'
import { requestLogger } from './middleware/log.ts'
import { healthRouter } from './routes/health.ts'
import { config } from './config.ts'

const app = express()

// Middleware (applied in order)
app.use(express.json())
app.use(requestLogger)          // SEC-05: token redaction before any logging

// Routes
app.use(healthRouter)

// Global error handler — Express 5 requires exactly 4 params
app.use((err: unknown, _req: express.Request, res: express.Response, _next: express.NextFunction) => {
  console.error('[error]', err)
  res.status(500).json({ error: 'Internal server error' })
})

// INFRA-03: Verify MinIO BEFORE listening (Pitfall 5: never reverse this order)
await verifyMinioConnection()

app.listen(config.port, () => {
  console.log(`[startup] DockGate listening on :${config.port}`)
})
```

Note: Bun supports top-level `await` in TypeScript modules — no IIFE wrapper needed.

Create three files:

**Dockerfile** (INFRA-02, INFRA-04):

Two stages: `deps` installs ALL packages (dev + prod in one pass then prod-only), `final` copies prod node_modules and source.

CRITICAL requirements from CLAUDE.md and research:
- Base image: `oven/bun:1` (Debian — NOT alpine variant; musl crash)
- ENTRYPOINT exec-form JSON array (NOT shell-form `sh -c ...`): Bun becomes PID 1, receives SIGTERM
- `USER bun` (non-root user included in oven/bun image)
- `EXPOSE 3000` (default port; actual port set via PORT env var at runtime)
- Copy `bun.lock*` with glob to handle both `bun.lock` and future `bun.lockb`

```dockerfile
# Stage 1: Install production dependencies
FROM oven/bun:1 AS deps
WORKDIR /app
COPY package.json bun.lock* ./
RUN bun install --frozen-lockfile --production

# Stage 2: Final image — production only
FROM oven/bun:1
WORKDIR /app

# Copy production node_modules from deps stage
COPY --from=deps /app/node_modules ./node_modules

# Copy source and config files
COPY src ./src
COPY package.json tsconfig.json ./

# Run as non-root bun user (included in oven/bun:1 image)
USER bun

# Document default port (actual port set via PORT env var)
EXPOSE 3000

# Exec-form ENTRYPOINT: Bun is PID 1, receives SIGTERM correctly
# Do NOT use shell-form: ENTRYPOINT ["sh", "-c", "bun run ..."] — breaks SIGTERM
ENTRYPOINT ["bun", "run", "src/index.ts"]
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| HTTP client→Express | All inbound HTTP requests — health endpoint is unauthenticated but stateless |
| container→MinIO | Network call from app container to MinIO container via Docker internal network |
| Portainer→container | SIGTERM sent to container on stop; must reach Bun (exec-form ENTRYPOINT) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | Elevation of Privilege | Dockerfile | mitigate | `USER bun` runs container as non-root; oven/bun:1 includes bun user with uid 1000 |
| T-04-02 | Denial of Service | health endpoint | accept | /health calls bucketExists() on every request; no rate limit in v1 (internal use only, Portainer health check frequency is low); SEC-V2-01 covers rate limiting |
| T-04-03 | Tampering | Dockerfile ENTRYPOINT | mitigate | Exec-form ENTRYPOINT ensures Bun is PID 1 and receives SIGTERM; shell-form would break signal forwarding (Pitfall 3) |
| T-04-04 | Information Disclosure | .env.example | mitigate | Only placeholder values committed; real .env is gitignored; .dockerignore excludes .env from build context |
| T-04-05 | Spoofing | health endpoint | accept | /health is unauthenticated by design (INFRA-01 — Portainer uses it without auth); no sensitive data returned |
</threat_model>
