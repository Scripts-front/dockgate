**POST /:name/upload handler (WRITE-01):**

Middleware chain: `validateAppName, requireUploadToken, validateVersion`
- validateVersion handles req.query.version — rejects missing/invalid version with 400

```typescript
appsRouter.post('/:name/upload', validateAppName, requireUploadToken, validateVersion, async (req, res) => {
  const { name } = req.params
  const version = req.query.version as string

  const rawUrl = await minioClient.presignedPutObject(config.minioBucket, tarKey(name, version), 900)  // D-02: 900s upload
  const url = rewritePresignedUrl(rawUrl)  // replace internal hostname with MINIO_PUBLIC_ENDPOINT
  res.json({ url })  // D-01: only { url: string }
})
```

**No statObject check before presignedPutObject** — this is intentional. The CI/CD pipeline
is uploading a NEW file. Checking existence first would break the workflow. The anti-phantom
check is only for PUT /latest (after the upload already happened).

**PUT /:name/latest handler (WRITE-02, WRITE-03):**

Middleware chain: `validateAppName, requireUploadToken`
- No validateVersion — version comes from body (D-05), not URL. Body validation done inline.

```typescript
const SHA256_REGEX = /^[a-f0-9]{64}$/i  // D-06

appsRouter.put('/:name/latest', validateAppName, requireUploadToken, async (req, res) => {
  const { name } = req.params
  const { version, sha256, size } = req.body as { version: unknown; sha256: unknown; size: unknown }

  // Input validation — ALL checks before anti-phantom (D-06 specifies 400 before MinIO call)
  if (typeof version !== 'string' || !version || !/^[a-zA-Z0-9._-]+$/.test(version) || version.includes('..')) {
    res.status(400).json({ error: 'Invalid version' })
    return
  }
  if (typeof sha256 !== 'string' || !SHA256_REGEX.test(sha256)) {
    res.status(400).json({ error: 'Invalid sha256' })  // D-06
    return
  }
  if (typeof size !== 'number' || !Number.isInteger(size) || size <= 0) {
    res.status(400).json({ error: 'Invalid size' })  // D-07
    return
  }

  // Anti-phantom check (WRITE-03, D-10) — verify .tar exists before writing latest.json
  try {
    await minioClient.statObject(config.minioBucket, tarKey(name, version))
  } catch (err) {
    if (isObjectNotFound(err)) {
      res.status(422).json({ error: `Tar file not found for version ${version}` })  // D-10
      return
    }
    throw err  // unexpected error — global handler returns 500
  }

  // Build and write latest.json (D-05, D-08)
  const manifest: LatestManifest = {
    schema: 1,
    version,
    sha256,
    size,
    publishedAt: new Date().toISOString(),  // D-08: server time, not from CI/CD
  }
  const json = JSON.stringify(manifest)
  await minioClient.putObject(
    config.minioBucket,
    latestKey(name),
    json,
    Buffer.byteLength(json),
    { 'Content-Type': 'application/json' },
  )

  res.status(200).json({ ok: true })
})
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CI/CD pipeline → POST /upload | Caller presents UPLOAD_TOKEN; receives presigned PUT URL to use directly against MinIO |
| CI/CD pipeline → PUT /latest | Caller presents UPLOAD_TOKEN + JSON body; API writes to MinIO after validation |
| API → MinIO (write path) | API uses MinIO client credentials to statObject and putObject; internal Docker network |
| MinIO presigned PUT URL → CI/CD | CI/CD puts .tar directly to MinIO; MinIO validates SigV4 on the presigned URL |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-08 | Spoofing | POST /upload + PUT /latest — token validation | mitigate | requireUploadToken uses timingSafeEqual(sha256(input), sha256(expected)) — constant-time comparison prevents timing oracle (SEC-02) |
| T-02-09 | Tampering | PUT /latest — version field in body used to construct MinIO key | mitigate | Inline validation: `typeof version === 'string' && /^[a-zA-Z0-9._-]+$/.test(version) && !version.includes('..')` before any MinIO call — prevents path traversal in tarKey/latestKey (SEC-03) |
| T-02-10 | Tampering | PUT /latest — sha256 field | mitigate | SHA256_REGEX = `/^[a-f0-9]{64}$/i` enforced before writing (D-06) — prevents injection of arbitrary strings into latest.json integrity field |
| T-02-11 | Tampering | PUT /latest — phantom version attack | mitigate | statObject check (WRITE-03) ensures .tar exists before latest.json is written — prevents publishing a version with no downloadable artifact |
| T-02-12 | Repudiation | POST /upload — presigned URL issued | accept | Presigned URL expires in 900s; if CI/CD does not complete upload, URL expires and no lasting object is created; audit trail via MinIO server logs if needed in future |
| T-02-13 | Information Disclosure | PUT /latest body | accept | Body contains only version string, sha256 hash, and size integer — no secrets or PII; UPLOAD_TOKEN validated before body is processed |
| T-02-14 | Denial of Service | PUT /latest — JSON body size | accept | express.json() default limit is 100kb; latest.json manifest body is ~200 bytes; effective limit is in place by default |
| T-02-15 | Elevation of Privilege | POST /upload — unrestricted upload path | accept | Presigned PUT URL is scoped to exact key `{appName}/{version}.tar` signed at generation time; MinIO enforces the scoped path; caller cannot PUT to a different key with this URL |
</threat_model>
