Create the directory and file:

```
mkdir -p /root/dockgate/.github/workflows
```

Create `.github/workflows/lint.yml` with the following exact content:

```yaml
name: Lint

on:
  push:
    branches:
      - master
  pull_request:
    branches:
      - master

jobs:
  lint:
    name: ESLint
    runs-on: ubuntu-latest

    steps:
      - name: Checkout
        uses: actions/checkout@v4

      - name: Setup Bun
        uses: oven-sh/setup-bun@v2
        with:
          bun-version: latest

      - name: Install dependencies
        run: bun install --frozen-lockfile

      - name: Run ESLint
        run: bun run lint
```

Notes:
- `actions/checkout@v4` is the current stable checkout action
- `oven-sh/setup-bun@v2` is the official Bun GitHub Actions action
- `bun install --frozen-lockfile` ensures CI uses exactly the versions in bun.lock (reproducible)
- `bun run lint` matches the script name added in Plan 01 Task 2
- Triggers: `push` and `pull_request` on `master` only (per D-05 and the specifics in CONTEXT.md)
- This workflow is for the DockGate repository itself — NOT the distribution pipeline (which lives in docs/examples/github-actions.yml per Phase 3 decision)

## Trust Boundaries

| GitHub Actions runner → repository | Workflow runs in ephemeral runner with read-only checkout |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-04 | Tampering | .github/workflows/lint.yml | mitigate | Workflow committed to git; changes require code review. No write permissions granted to lint job — read-only checkout is sufficient |
| T-04-05 | Elevation of Privilege | workflow permissions | mitigate | Lint workflow requires no secrets and no write permissions. Default GITHUB_TOKEN read-only scope is sufficient. No `permissions:` block needed — GitHub defaults to minimal permissions on public repos |
