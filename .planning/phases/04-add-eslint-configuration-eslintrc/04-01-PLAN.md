  <action>
Install ESLint and typescript-eslint as dev dependencies:

```
bun add -d eslint typescript-eslint
```

This installs ESLint v9 and the `typescript-eslint` unified package (which includes the parser, plugin, and config presets in one).

Create `eslint.config.js` in the project root with the following exact content:

```js
// @ts-check
import tseslint from "typescript-eslint";

export default tseslint.config(
  // Target ONLY src/**/*.ts — config files at root are excluded (per D-07)
  { files: ["src/**/*.ts"] },

  // typescript-eslint/strict preset — more rigorous than recommended,
  // no type-checking overhead (no parserOptions.project) (per D-02, D-03)
  ...tseslint.configs.strict,

  // Discretion: enforce import type for type-only imports
  // Required because tsconfig has verbatimModuleSyntax: true
  {
    rules: {
      "@typescript-eslint/consistent-type-imports": [
        "error",
        { prefer: "type-imports", fixStyle: "inline-type-imports" }
      ],
      // Discretion: warn on console usage (not an error — health route uses it)
      "no-console": "warn",
    },
  }
);
```

Note on `tseslint.configs.strict`: this is the non-type-checked variant. It does NOT require `parserOptions.project` and has no performance overhead from type analysis (per D-03). The `typescript-eslint` unified package (v8+) exports `strict` directly from `tseslint.configs`.

Note on `eslint.config.js` extension: the project `package.json` does NOT have `"type": "module"`, so using `import` syntax requires either adding `"type": "module"` to package.json OR naming the file `eslint.config.mjs`. However, since Bun is the runtime and Bun handles ES modules natively regardless of package.json type field, the `eslint.config.js` with ES module syntax works when ESLint is invoked via `bun run lint`. If ESLint complains about ES module syntax, rename to `eslint.config.mjs` — same content, just different extension.

The safer approach: create `eslint.config.mjs` if ESLint cannot parse `eslint.config.js` as ES module. Prefer `eslint.config.js` first; fall back to `.mjs` if needed.
